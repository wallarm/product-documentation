Environment variable | Description | Required
--- | ---- | ----
`WALLARM_API_TOKEN` | Wallarm node veya API token'ı. | Yes
`WALLARM_LABELS` | <p>Node 4.6'dan itibaren mevcuttur. Sadece `WALLARM_API_TOKEN` [API token][api-token] değeri `Deploy` rolü ile ayarlandığında çalışır. Node örneği gruplaması için `group` etiketini ayarlar, örneğin:</p> <p>`WALLARM_LABELS="group=<GROUP>"`</p> <p>...node örneği, `<GROUP>` örnek grubuna yerleştirilecektir (varsa mevcut, yoksa oluşturulacaktır).</p> | Yes (for API tokens)
`NGINX_BACKEND` | Wallarm çözümüyle korunacak kaynak için alan adı veya IP adresi. | Yes
`WALLARM_API_HOST` | Wallarm API sunucusu:<ul><li>`us1.api.wallarm.com` for the US Cloud</li><li>`api.wallarm.com` for the EU Cloud</li></ul>Varsayılan: `api.wallarm.com`. | No
`WALLARM_MODE` | Node modu:<ul><li>`block`: kötü amaçlı istekleri engellemek için</li><li>`safe_blocking`: yalnızca [graylisted IP addresses][graylist-docs] kaynaklı kötü amaçlı istekleri engellemek için</li><li>`monitoring`: istekleri analiz etmek ancak engellememek için</li><li>`off`: trafik analizini ve işlemeyi devre dışı bırakmak için</li></ul>Varsayılan: `monitoring`.<br>[Filtrasyon modlarının detaylı açıklaması →][filtration-modes-docs] | No
`WALLARM_APPLICATION` | Wallarm Cloud'da kullanılacak, korunan uygulamanın benzersiz tanımlayıcısı. Değer, `0` dışındaki pozitif bir tamsayı olabilir.<br><br>Varsayılan değer (değişken konteynıra aktarılmamışsa) `-1` olup, Wallarm Console → **Settings** → **Application** bölümünde görüntülenen **default** uygulamayı belirtir.<br><br>[Uygulama yapılandırmasıyla ilgili daha fazla bilgi →][application-configuration] | No
`TARANTOOL_MEMORY_GB` | Tarantool için ayrılan [bellek miktarı][allocating-memory-guide]. Değer bir tamsayı veya ondalık sayı olabilir (ondalık ayırıcı olarak nokta <code>.</code> kullanılır). Varsayılan: 1 gigabayt. | No
`NGINX_PORT` | NGINX'in Docker konteynırı içinde kullanacağı portu ayarlar.<br><br>Docker imajı `4.0.2-1`'den itibaren, [`wallarm-status`][node-status-docs] servisi otomatik olarak NGINX ile aynı portu kullanır.<br><br>Varsayılan değer (değişken konteynıra aktarılmamışsa) `80`.<br><br>Sözdizimi: `NGINX_PORT='443'`. | No
`WALLARM_STATUS_ALLOW` | Docker konteynırının dışından [`/wallarm-status` endpoint][node-status-docs]'ine erişimine izin verilen özel CIDR’ler. Örnek değer: `10.0.0.0/8`. Birden çok değer iletmeniz gerekiyorsa, ayırıcı olarak virgül `,` kullanın. Servise dışarıdan erişmek için, Docker konteynırının IP adresini kullanarak `/wallarm-status` endpoint yolunu belirtin. | No
`DISABLE_IPV6` | Boş olmayan herhangi bir değer içeren değişken, NGINX yapılandırma dosyasından `listen [::]:80 default_server ipv6only=on;` satırını siler; bu da NGINX'in IPv6 bağlantılarının işlenmesini durdurur.<br><br>Eğer değişken açıkça belirtilmemişse veya boş bir değer `""` ise, NGINX hem IPv6 hem de IPv4 bağlantılarını işler. | No
