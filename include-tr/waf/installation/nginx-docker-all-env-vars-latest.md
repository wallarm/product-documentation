```markdown
Ortam değişkeni | Açıklama | Gereklilik
--- | --- | ---
`WALLARM_API_TOKEN` | Wallarm node veya API belirteci. | Evet
`WALLARM_LABELS` | <p>Düğüm 4.6'dan itibaren kullanılabilir. Sadece `WALLARM_API_TOKEN` değeri `Deploy` rolüyle ayarlanmış [API token][api-token] ile çalışır. Node örneği gruplaması için `group` etiketini ayarlar, örneğin:</p> <p>`WALLARM_LABELS="group=<GROUP>"`</p> <p>...node örneğini `<GROUP>` örnek grubuna (varsa mevcut; yoksa oluşturularak) yerleştirir.</p> | Evet (API token'lar için)
`NGINX_BACKEND` | Wallarm çözümü ile korunacak kaynağın domain veya IP adresi. | Evet
`WALLARM_API_HOST` | Wallarm API sunucusu:<ul><li>ABD Cloud için `us1.api.wallarm.com`</li><li>EU Cloud için `api.wallarm.com`</li></ul>Varsayılan: `api.wallarm.com`. | Hayır
`WALLARM_MODE` | Düğüm modu:<ul><li>`block` – zararlı istekleri engellemek için</li><li>`safe_blocking` – yalnızca [graylisted IP addresses][graylist-docs] kaynaklı zararlı istekleri engellemek için</li><li>`monitoring` – istekleri analiz edip engellememek için</li><li>`off` – trafik analizini ve işlenmesini devre dışı bırakmak için</li></ul>Varsayılan: `monitoring`.<br>[Filtrasyon modlarının ayrıntılı açıklaması →][filtration-modes-docs] | Hayır
`WALLARM_APPLICATION` | Wallarm Cloud'da kullanılacak korunan uygulamanın benzersiz tanımlayıcısı. Değer, `0` hariç pozitif bir tamsayı olabilir.<br><br>Varsayılan değer (değişken konteynere aktarılmazsa) `-1`'dir ve bu, Wallarm Console → **Settings** → **Application**'da gösterilen **varsayılan** uygulamayı belirtir.<br><br>[Uygulamaların yapılandırılması hakkında daha fazla bilgi →][application-configuration] | Hayır
`TARANTOOL_MEMORY_GB` | Tarantool'a ayrılan [bellek miktarı][allocating-memory-guide]. Değer, bir tamsayı veya ondalık sayı (ondalık ayırıcı olarak nokta <code>.</code>) olabilir. Varsayılan: 1 gigabayt. | Hayır
`NGINX_PORT` | Docker konteyneri içinde NGINX'in kullanacağı portu ayarlar.<br><br>Docker imajı `4.0.2-1`'den itibaren, [`wallarm-status`][node-status-docs] servisi otomatik olarak NGINX ile aynı portta çalışır.<br><br>Varsayılan değer (değişken konteynere aktarılmazsa) `80`'dir.<br><br>Sözdizimi: `NGINX_PORT='443'`. | Hayır
<a name="wallarm-status-allow-env-var"></a>`WALLARM_STATUS_ALLOW` | Docker konteyneri dışından [`/wallarm-status` uç noktası][node-status-docs]'na erişime izin verilen özel CIDR'ler. Örnek değer: `10.0.0.0/8`. Birden fazla değer geçirmeniz gerekiyorsa, ayırıcı olarak virgül `,` kullanın. Hizmete dışarıdan erişmek için, Docker konteynerinin IP'sini kullanarak `/wallarm-status` uç nokta yolunu belirtin. | Hayır
`DISABLE_IPV6` | Boş olmayan herhangi bir değere sahip değişken, NGINX yapılandırma dosyasından `listen [::]:80 default_server ipv6only=on;` satırını siler; böylece NGINX'in IPv6 bağlantı işlemesi durur.<br><br>Eğer değişken açıkça belirtilmez veya boş değere `""` sahipse, NGINX hem IPv6 hem de IPv4 bağlantılarını işler. | Hayır
`WALLARM_APIFW_ENABLE` | Bu ayar, 4.10 sürümünden itibaren kullanılabilen [API Specification Enforcement][api-policy-enf-docs]'ı etkinleştirip devre dışı bırakır. Bu özelliğin etkinleştirilmesi, gerekli abonelik ve Wallarm Console UI üzerinden yapılandırmanın yerine geçmez.<br><br>Varsayılan değeri, işlevselliği etkinleştiren `true`'dur. | Hayır
```
